# Deployment Guide

This app deploys as two services, **both free, both requiring no credit card**:

| Service | Platform | Purpose |
|---|---|---|
| Frontend | Vercel | TanStack Start React app |
| Backend API | Render (free web service) | Django + Daphne ASGI |
| Database | Neon (free PostgreSQL) | Persistent data store |

## Quick Start

### 1. Create Neon Database (free, no CC)

1. Go to **[neon.tech](https://neon.tech)** → sign up with GitHub
2. Create project: name = `campaign-copilot`, database = `campaign_copilot`
3. Copy the **connection string** from the Connection Details panel

### 2. Deploy Backend on Render (free, no CC)

1. Go to **[render.com](https://render.com)** → sign up with GitHub
2. **New → Blueprint** → select your `campaign-copilot` repo
3. Render auto-detects `render.yaml` — expand the service and fill in:
   - `DATABASE_URL` → paste your Neon connection string
   - `GEMINI_API_KEY` → your key from [aistudio.google.com](https://aistudio.google.com/app/apikey)
4. Click **Apply** and wait ~5 min for build + seed

Verify: `https://campaign-copilot-api.onrender.com/api/stats/` should return `{"total_customers": 1000, ...}`

### 3. Configure Vercel Frontend

In Vercel → your project → Settings → Environment Variables, add:

```
VITE_API_BASE_URL = https://campaign-copilot-api.onrender.com
VITE_WS_BASE_URL = wss://campaign-copilot-api.onrender.com
```

Redeploy Vercel (or push a commit). Done.

## Environment Variables Reference

### Render (backend)
| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | ✅ | Neon PostgreSQL connection string |
| `GEMINI_API_KEY` | ✅ | For AI Copilot features |
| `DJANGO_SECRET_KEY` | ✅ | Auto-generated by Render |
| `DJANGO_DEBUG` | — | Set to `false` (default in render.yaml) |
| `CORS_ALLOW_ALL` | — | `true` during setup, set `FRONTEND_URL` then `false` |
| `FRONTEND_URL` | recommended | Your Vercel URL for strict CORS |

### Vercel (frontend)
| Variable | Required | Description |
|---|---|---|
| `VITE_API_BASE_URL` | ✅ | Full URL of your Render backend |
| `VITE_WS_BASE_URL` | optional | WebSocket URL (defaults to VITE_API_BASE_URL with ws://) |

## Local Development

Backend still works locally unchanged:
```sh
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data --customers 1000 --orders 5000
python manage.py rfm_compute
python manage.py build_segments
python manage.py runserver
```

Frontend:
```sh
npm install
npm run dev   # Uses http://127.0.0.1:8000 if VITE_API_BASE_URL is unset
```
